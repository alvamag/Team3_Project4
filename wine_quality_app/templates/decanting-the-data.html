Decanting the Data

Here, we outline how our machine learning model was trained and tested to predict wine quality, including the dataset and methodologies used.

Wine Quality Prediction Using Supervised Learning

  • Standard vs. Premium
  • Dataset: Merged Red and White Wines
  • Tools: Python, Spark SQL, Scikit-learn

Data Cleaning & Feature Engineering

  • Merged two datasets (Red & White)
  • Removed nulls and standardized feature types
  • Applied feature scaling (standardization + normalization)
  • Saved cleaned data as cleaned_wine_data_with_scaling.csv

Spark SQL Analysis

  • White wines scored slightly higher on average
  • Premium wines had higher alcohol and lower density
  • Spark SQL confirmed strong feature patters before modeling

Model Preparation & Lable Binning

  • Quality scores converted to two labels: Standard (≤ 6), Premium (> 6)
  • Target labels created using a custom function
  • Categorical features (like wine type) one-hot encoded
  • Scaled features used as input for ML models

Initial Model Training

  • Used Random Forest Classifier as the baseline model
  • Train/test split: 80% training, 20% testing
  • Target: Predict Standard vs. Premium wines
  • Initial accuracy: 89.92% (pre-optimization)

Model Optimization

  • Models: Random Forest, Logistic Regression, SVM
  • Trained with full features and PCA-reduced features
  • PCA used to reduce dimensionality and simplify inut
  • Compared results with and without PCA

Model Evaluation

  • Best Model: Random Forest (Accuracy: 89.46%)
  • Premium wines harder to predict due to class imbalance
  • PCA models performed slightly worse
  • Metrics Used: Accuracy, Precision, Recall, F1-Score

Feature Importances

  • Feature importances show how much each variable contributes to making accurate predictions/li>
  • In a Random Forest, features that split the data more cleanly are ranked higher
  • Think of it like asking the model, “Which ingredients matter most when choosing a premium wine?
  • These scores aren’t based on correlation—they reflect how useful each feature was during training
  • It gives us a peek into the model’s “thought process,” not just its output
